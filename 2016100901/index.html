为什么只有2、8、16进制，没有其它进制

Posted by Float8 on Sunday, October 9, 2016

编程过程中我们常用的还是10进制数，必竟10进制是人类生活生产中常用的进制，在编程中自然也是最常用的进制。

为什么需要2进制、8进制和16进制?

为什么需要2进制

1、计算机是由逻辑电路组成，逻辑电路通常只有两个状态，开关的接通与断开，这两种状态正好可以用“1”和“0”表示；

2、二进制中只使用0和1两个数字，传输和处理时不易出错，因而可以保障计算机具有很高的可靠性。

为什么需要8进制和16进制?

由于数据在计算机中的表示，最终以2进制的形式存在，所以有时候在计算机中使用2进制，可以更直观地解决问题。

但2进制数太长了。比如int类型占用4个字节，32位。比如100，用int类型的二进制数表达将是：

0000 0000 0000 0000 0110 0100

面对这么长的数大大增加了进行思考和操作的难度，在编程上造成了非常大的困难。

用16进制或8进制可以解决这个问题。因为，进制越大，数的表达长度也就越短。不过，为什么偏偏是16或8进制，而不其它的，诸如9或20进制呢？

2、8、16，分别是2的1次方，3次方，4次方。这一点使得三种进制之间可以非常直接地互相转换。8进制或16进制缩短了2进制数，但保持了2进制数的表达特点。

转10进制数

2进制数转换为10进制数

2进制就是逢2进1,用0,1表达。2进制数第0位的权值是2^0，第1位的权值是2^1……，我们可以得到如下公式：

d：代表十进制数； b：代表二进制数1或0； n：代表第几位。

d = b * 2^n + b * 2^(n+1) + b * 2^(n+2) + b * 2^(n+..)+…

设有一个2进制数0110 0100，转换为10进制为

先把每一位转为10进制数

第0位 0 * 2^0 = 0
第1位 0 * 2^1 = 0
第2位 1 * 2^2 = 4
第3位 0 * 2^3 = 0
第4位 0 * 2^4 = 0
第5位 1 * 2^5 = 32
第6位 1 * 2^6 = 64
第7位 0 * 2^7 = 0

求每位的相加之和就是10进制数

0 * 2^0 + 0 * 2^1 + 1 * 2^2 + 1 * 2^3 + 0 * 2^4 + 1 * 2^5 + 1 * 2^6 + 0 * 2^7 = 100

0乘以多少都是0，所以我们也可以直接跳过值为0的位：

1 * 2^2 + 1 * 2^3 + 1 * 2^5 + 1 * 2^6 = 100

8进制数转换为10进制数

8进制就是逢8进1。8进制数采用0～7这八数来表达1位数。表达8进制数，前面加0

8进制数第0位的权值为8^0，第1位权值为8^1，第2位权值为8^2……，我们可以得到如下公式：

d：代表十进制数； b：代表二进制数1或0； n：代表第几位。

d = b * 8^n + b * 8^(n+1) + b * 8^(n+2) + b * 8^(n+..)+…

设有一个8进制数：1507，转换为10进制为：

先把每一位转为10进制数

第0位 7 * 8^0 = 7
第1位 0 * 8^1 = 0
第2位 5 * 8^2 = 320
第3位 1 * 8^3 = 512

求每位的相加之和就是10进制数:

7 * 8^0 + 0 * 8^1 + 5 * 8^2 + 1 * 8^3 = 839

16进制数转换成10进制数

2进制，用两个阿拉伯数字：0、1；

8进制，用八个阿拉伯数字：0、1、2、3、4、5、6、7；

10进制，用十个阿拉伯数字：0到9；

16进制，用十六个阿拉伯数字……等等，阿拉伯人或说是印度人，只发明了10个数字啊？

16进制就是逢16进1，但我们只有0~9这十个数字，所以我们用A，B，C，D，E，F这五个字母来分别表示10，11，12，13，14，15。字母不区分大小写。表达8进制数，前面加0x

十六进制数的第0位的权值为16^0，第1位的权值为16^1，第2位的权值为16^2……，我们可以得到如下公式：

d：代表十进制数； b：代表二进制数1或0； n：代表第几位。

d = b * 16^n + b * 16^(n+1) + b * 16^(n+2) + b * 16^(n+..)+…

设有一个8进制数：2AF5，转换为10进制为：

先把每一位转为10进制数（A表示10，而F表示15)）

第0位： 5 * 160 = 5
第1位： F * 161 = 240
第2位： A * 162 = 2560
第3位： 2 * 163 = 8192

求每位的相加之和就是10进制数:

5 * 160 + F * 161 + A * 162 + 2 * 163 = 10997

10进制数转换到2、8、16进制数

10进制数转换为2进制数

10进制数转换成2进制数，这是一个连续除2的过程：

把要转换的数除以2得到商和余数，将商继续除以2直到商为0。最后将所有余数倒序排列，得到数就是转换结果。

设有一个10进制数6，转换为2进制为：

被除数 计算过程 商 余数
6 6 / 2 3 0
3 3 / 2 1 1
1 1 / 2 0 1

那么转换结果为110。

10进制数转换为8进制数

10进制数转换成8进制的方法，和转换为2进制的方法类似，惟一变化：除数由2变成8。

10进制数转换成8进制数，这是一个连续除8的过程：

把要转换的数除以8得到商和余数，将商继续除以8直到商为0。最后将所有余数倒序排列，得到数就是转换结果。

设有一个10进制数120，转换为8进制为：

被除数 计算过程 商 余数
20 120 / 8 15 0
15 15 / 8 1 7
1 1 / 8 0 1

那么转换结果为170。

10进制数转换为16进制数

10进制数转换成16进制的方法，和转换为2、8进制的方法类似，惟一变化：除数由2变成16。

10进制数转换成16进制数，这是一个连续除16的过程：

设有一个10进制数120，转换为16进制为：

被除数 计算过程 商 余数
120 120 / 16 7 8
7 7 / 16 0 7

那么转换结果为78

2、16进制数互相转换

我们使用16进制的最大的单位数(只占一位的数)0xf转换成二进制是1111。那么我们可以看到每位16进制数字是多少：

  • 第1个1为1
  • 第2个1为2
  • 第3个1为4
  • 第4个1为8

上述数字组合相加不存在数字不一样相加和相等的情况。接下来我们看两种进制的相互转化情况。

16进制转化为2进制

上面我们看到了最大单位数0xf转化为二进制就是4个1。当16进制数为多位的情况下，我们就可以从右至左把每位16进制的单位数换算成4位的2进制数直至到最后一位16进制数为止。

那么我们通过二进制的4位数的每个1代表的数16进制数字是不是就能够轻易转化了呢。只要我们通过8、4、2、1做个15以内（含15）的加减法运算即可。

接下来我们把小于等于0xf的数字做个二进制转化：

0xf = 15 = 8+4+2+1 = 1111
0xe = 14 = 8+4+2+0 = 1110
0xd = 13 = 8+4+0+1 = 1101
0xc = 12 = 8+4+0+0 = 1100
0xb = 11 = 8+0+2+1 = 1011
0xa = 10 = 8+0+2+0 = 1010
0x9 =  9 = 8+0+0+1 = 1001
0x8 =  8 = 8+0+0+0 = 1000
0x7 =  7 = 0+4+2+1 = 0111
0x6 =  6 = 0+4+2+0 = 0110
0x5 =  5 = 0+4+0+1 = 0101
0x4 =  4 = 0+4+0+0 = 0100
0x3 =  3 = 0+0+2+1 = 0011
0x2 =  2 = 0+0+2+0 = 0010
0x1 =  1 = 0+0+0+1 = 0001

从上面的15（包含15）以内的16进制转2进制是不是非常容易呢

2进制转化为16进制

从右边开始，每四位划分

1111  = 8+4+2+1 = 15 = 0xf
1110  = 8+4+2+0 = 14 = 0xe
1101  = 8+4+0+1 = 13 = 0xd
1100  = 8+4+0+0 = 12 = 0xc
1011  = 8+0+2+1 = 11 = 0xb
1010  = 8+0+2+0 = 10 = 0xa
1001  = 8+0+0+1 =  9 = 0x9
1000  = 8+0+0+0 =  8 = 0x8
0111  = 0+4+2+1 =  7 = 0x7
0110  = 0+4+2+0 =  6 = 0x6
0101  = 0+4+0+1 =  5 = 0x5
0100  = 0+4+0+0 =  4 = 0x4
0011  = 0+0+2+1 =  3 = 0x3
0010  = 0+0+2+0 =  2 = 0x2
0001  = 0+0+0+1 =  1 = 0x1

那么我们看2进制转化为16进制是不是也特别容易。
